TRD-07 Hints

Progressive Guidance — Conduit HTTP Server Project
Level 1 = nudge  |  Level 2 = stronger push

H1 — Running wrk Benchmarks (D1)

Level 1

Run wrk 4 times, increasing -c each time: 1, 10, 50, 100. Use 2 threads (-t2) and 10-second duration (-d10s). The target URL should be a real file in the docroot, not just / — use /index.html so the full file-serving path is exercised. Do a 5-second warm-up run first that you discard.

Level 2
# Warm up (discard)
wrk -t2 -c10 -d5s http://localhost:9876/index.html > /dev/null

# Record benchmarks
for CONNS in 1 10 50 100; do
    echo "=== Concurrency: $CONNS ==="
    wrk -t2 -c$CONNS -d10s --latency http://localhost:9876/index.html
    echo ""
done

The --latency flag adds percentile breakdown (p50, p75, p90, p99). Extract these for the README table. Run each 3 times and take the median req/sec and p99 values.

H2 — Valgrind Memcheck (D3)

Level 1

Start the server under valgrind, send requests that exercise all code paths (200, 404, 405, 400, large file), then send SIGTERM and read valgrind's output. The key line is "definitely lost" — it must be 0. "Still reachable" is usually fine at process exit.

Level 2
# Terminal 1: start under valgrind
valgrind --leak-check=full --show-leak-kinds=all \
    ./conduit 9876 ./docroot 2> valgrind_out.txt &
VG_PID=$!
sleep 2  # valgrind startup is slow

# Terminal 2: exercise all code paths
curl -s http://localhost:9876/
curl -s http://localhost:9876/nonexistent
curl -s -X POST http://localhost:9876/
echo -ne "GARBAGE\r\n\r\n" | nc -w 1 localhost 9876
curl -s http://localhost:9876/index.html

# Graceful shutdown
kill -TERM $VG_PID
wait $VG_PID 2>/dev/null || true

# Check results
grep "definitely lost" valgrind_out.txt

If "definitely lost" is non-zero, valgrind shows the allocation stack trace. Work backwards from there to find the missing free().
