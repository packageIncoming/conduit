TRD-06 Hints

Progressive Guidance — Conduit HTTP Server Project
Level 1 = nudge  |  Level 2 = stronger push  |  15+ minutes before escalating

H1 — SIGPIPE Suppression (R1)

Level 1

One line at the very start of main(), before any sockets are created. After this, writing to a broken pipe returns an error instead of terminating the process. Every write error path in the server must handle EPIPE the same way it handles any other write failure: close the connection and move on.

Level 2
signal(SIGPIPE, SIG_IGN);

That's it. After this, write(fd, ...) returns -1 with errno == EPIPE when the remote end has closed. The existing write error handling from TRD-04/05 (close connection on write failure) covers this automatically — as long as you actually check write return values everywhere.

H2 — Request Size Limit (R2)

Level 1

The check already exists implicitly in TRD-04's read handler: if read_len reaches the buffer capacity without finding \r\n\r\n, there's no more room. The change for TRD-06 is making this explicit — send a 400 response before closing, rather than silently dropping the connection.

Level 2

In the read handler, after checking buffer space:

int space = sizeof(conn->read_buf) - conn->read_len - 1;
if (space <= 0) {
    /* Buffer full, no complete headers found */
    http_response_t resp = {0};
    response_set_status(&resp, 400, "Bad Request");
    response_set_body(&resp, "Request too large", 17, 0);
    response_add_header(&resp, "Content-Type", "text/plain");
    /* Write directly — best effort, then close */
    response_send(conn->fd, &resp);
    response_cleanup(&resp);
    close_connection(epfd, conn);
    return;
}

H3 — Idle Connection Timeout (R4)

Level 1

Three changes: (1) add a time_t last_active field to connection_t, set it on accept and on every successful read/write. (2) Change epoll_wait() from blocking forever (-1) to a timeout (e.g., 1000 ms). (3) After epoll_wait() returns, sweep all connections and close any where time(NULL) - last_active exceeds the threshold.

Level 2

The sweep needs access to all active connections. Maintain a list (array or linked list) of all connection_t * pointers. After processing events:

#define IDLE_TIMEOUT 30  /* seconds */

/* In event loop, after processing events: */
time_t now = time(NULL);
for (int i = 0; i < active_count; i++) {
    if (connections[i] &&
        now - connections[i]->last_active > IDLE_TIMEOUT) {
        close_connection(epfd, connections[i]);
        connections[i] = NULL;
    }
}

Use clock_gettime(CLOCK_MONOTONIC, ...) instead of time() for a clock that's immune to wall-clock adjustments (e.g., NTP jumps). For Conduit, time() is sufficient.
