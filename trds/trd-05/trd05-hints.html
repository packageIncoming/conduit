TRD-05 Hints

Progressive Guidance — Conduit HTTP Server Project
Level 1 = nudge  |  Level 2 = stronger push  |  15+ minutes before escalating

H1 — Thread Pool Struct and Init (R1)

Level 1

The pool struct needs: an array of pthread_t thread IDs, a mutex, a condvar, a pointer to the task queue head/tail, a queue size counter, a shutdown flag, and the thread count. threadpool_init() initializes the mutex/condvar, sets shutdown to 0, and spawns N threads that all run the same worker function.

Level 2
typedef struct {
    pthread_t *threads;
    int thread_count;
    task_node_t *queue_head;
    task_node_t *queue_tail;
    int queue_size;
    pthread_mutex_t mutex;
    pthread_cond_t cond;
    int shutdown;
} threadpool_t;

threadpool_t *threadpool_init(int n) {
    threadpool_t *pool = calloc(1, sizeof(threadpool_t));
    pool->threads = malloc(sizeof(pthread_t) * n);
    pool->thread_count = n;
    pthread_mutex_init(&pool->mutex, NULL);
    pthread_cond_init(&pool->cond, NULL);
    for (int i = 0; i < n; i++) {
        pthread_create(&pool->threads[i], NULL, worker_fn, pool);
    }
    return pool;
}

H2 — Task Queue (R2)

Level 1

A singly linked list is simplest. Each node holds a connection_t * and a next pointer. Enqueue appends to the tail, dequeue removes from the head. Both operations must happen under the mutex. After enqueue, signal the condvar to wake one worker.

Level 2
typedef struct task_node {
    connection_t *conn;
    struct task_node *next;
} task_node_t;

void threadpool_enqueue(threadpool_t *pool, connection_t *conn) {
    task_node_t *node = malloc(sizeof(task_node_t));
    node->conn = conn;
    node->next = NULL;

    pthread_mutex_lock(&pool->mutex);
    if (pool->queue_tail) {
        pool->queue_tail->next = node;
    } else {
        pool->queue_head = node;
    }
    pool->queue_tail = node;
    pool->queue_size = pool->queue_size + 1;
    pthread_cond_signal(&pool->cond);
    pthread_mutex_unlock(&pool->mutex);
}

H3 — Worker Thread Function (R3)

Level 1

The worker function takes the pool pointer as its argument. It loops forever: lock, wait on condvar while the queue is empty and shutdown is not set, dequeue a task, unlock, process the task. If shutdown is set and the queue is empty, the worker returns (exits the thread).

Level 2
void *worker_fn(void *arg) {
    threadpool_t *pool = (threadpool_t *)arg;

    while (1) {
        pthread_mutex_lock(&pool->mutex);

        while (pool->queue_size == 0 && !pool->shutdown) {
            pthread_cond_wait(&pool->cond, &pool->mutex);
        }

        if (pool->shutdown && pool->queue_size == 0) {
            pthread_mutex_unlock(&pool->mutex);
            return NULL;
        }

        /* Dequeue */
        task_node_t *node = pool->queue_head;
        pool->queue_head = node->next;
        if (!pool->queue_head) pool->queue_tail = NULL;
        pool->queue_size = pool->queue_size - 1;

        pthread_mutex_unlock(&pool->mutex);

        /* Process: parse request, serve file, send response */
        process_request(node->conn);
        close(node->conn->fd);
        free(node->conn);
        free(node);
    }
    return NULL;
}

Note: process_request() encapsulates TRD-01 parsing + TRD-02 file serving + TRD-03 response engine. The worker does everything after the raw bytes have been read.

H4 — Graceful Shutdown (R6)

Level 1

Three steps in order: set the flag, wake everyone, wait for everyone. If you broadcast before setting the flag, workers wake up, see no flag, and go back to sleep. If you set the flag but don't broadcast, sleeping workers never check it.

Level 2
void threadpool_destroy(threadpool_t *pool) {
    pthread_mutex_lock(&pool->mutex);
    pool->shutdown = 1;
    pthread_cond_broadcast(&pool->cond);
    pthread_mutex_unlock(&pool->mutex);

    for (int i = 0; i < pool->thread_count; i++) {
        pthread_join(pool->threads[i], NULL);
    }

    /* Free any remaining queued tasks */
    task_node_t *node = pool->queue_head;
    while (node) {
        task_node_t *next = node->next;
        close(node->conn->fd);
        free(node->conn);
        free(node);
        node = next;
    }

    pthread_mutex_destroy(&pool->mutex);
    pthread_cond_destroy(&pool->cond);
    free(pool->threads);
    free(pool);
}
Thread safety note for TRD-01 code: If the request parser uses strtok(), it will break under threading — strtok() uses global state. This is why the TRD-01 Prior Knowledge doc recommended strtok_r(). If strtok() was used, now is the time to replace it.
