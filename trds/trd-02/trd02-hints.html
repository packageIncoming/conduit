TRD-02 Hints

Progressive Guidance — Conduit HTTP Server Project
Level 1 = nudge  |  Level 2 = stronger push  |  15+ minutes before escalating

H1 — Path Resolution (R1)

Level 1

You need to join two strings: the document root and the request path. Think about what character separates them. What if the docroot ends with / and the request path starts with /? What if neither has one?

Level 2

Use snprintf() to build the combined path. If the request path is /, append index.html instead:

char filepath[2048];
if (strcmp(req.path, "/") == 0) {
    snprintf(filepath, sizeof(filepath), "%s/index.html", docroot);
} else {
    snprintf(filepath, sizeof(filepath), "%s%s", docroot, req.path);
}

The request path from TRD-01 already starts with /, so docroot + path naturally forms a valid path like /srv/www/style.css.

H2 — Directory Traversal Prevention (R2)

Level 1

Two steps: resolve the real path, then check it starts where it should. realpath() turns any path into its absolute canonical form. After resolving, a simple strncmp() against the docroot tells you if the file is inside the allowed directory.

Level 2
char resolved[PATH_MAX];
char resolved_root[PATH_MAX];

/* Resolve both the file path and the docroot */
if (realpath(filepath, resolved) == NULL) {
    /* File doesn't exist → 404 */
    return send_error(client_fd, 404);
}
realpath(docroot, resolved_root);

/* Prefix check */
if (strncmp(resolved, resolved_root, strlen(resolved_root)) != 0) {
    /* Path escaped docroot → 403 */
    return send_error(client_fd, 403);
}

Resolve the docroot too — don't compare a resolved path against a potentially relative docroot string. Both must be canonical.

H3 — File Reading (R3)

Level 1

Open the file, ask the OS how big it is, allocate a buffer that size, read into it. Don't forget: read() can return fewer bytes than you asked for, even on regular files. Loop until you've read st_size bytes or hit an error.

Level 2
int fd = open(resolved, O_RDONLY);
if (fd == -1) {
    /* errno == ENOENT → 404, else → 500 */
}

struct stat sb;
fstat(fd, &sb);

char *body = malloc(sb.st_size);
int total = 0;
while (total < sb.st_size) {
    int n = read(fd, body + total, sb.st_size - total);
    if (n <= 0) break;
    total += n;
}
close(fd);

/* Use sb.st_size for Content-Length */
/* Free body after sending the response */

H4 — MIME Type Mapping (R4)

Level 1

Find the last dot in the filename. Compare the extension string against known types. strrchr() finds the last occurrence of a character in a string — use it to find the dot.

Level 2
const char *get_mime_type(const char *path) {
    const char *ext = strrchr(path, '.');
    if (!ext) return "application/octet-stream";
    if (strcmp(ext, ".html") == 0) return "text/html";
    if (strcmp(ext, ".css") == 0)  return "text/css";
    if (strcmp(ext, ".js") == 0)   return "application/javascript";
    if (strcmp(ext, ".txt") == 0)  return "text/plain";
    if (strcmp(ext, ".jpg") == 0)  return "image/jpeg";
    if (strcmp(ext, ".png") == 0)  return "image/png";
    return "application/octet-stream";
}

This is simple and correct. A hash map would be faster but is unnecessary for 6 entries.
