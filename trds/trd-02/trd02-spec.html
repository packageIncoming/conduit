TRD-02: Static File Serving

Conduit HTTP Server Project
Depends on: TRD-01  |  Difficulty: Medium  |  Estimated: 4–6 sessions

Objective

Serve files from a document root directory. Map URI paths to filesystem paths, read file contents, set correct MIME types, and block directory traversal attacks.

Interface Change

./conduit <port> <docroot>

Both arguments required. Exit with code 1 and a usage message to stderr if either is missing.

Technical Requirements

R1 — Path Resolution

Construct a filesystem path from the document root and the request URI. GET / maps to <docroot>/index.html. All other paths map to <docroot> + path.

Stuck? → Hints H1

R2 — Directory Traversal Prevention

Resolve the constructed path with realpath(). Verify the resolved path begins with the document root prefix. If it does not, return 403 Forbidden. Do not rely on checking for .. in the raw URI.

Stuck? → Hints H2

R3 — File Reading

Open the resolved file with open(). Get its size with fstat(). Read the full contents into a buffer. Close the fd. If open() fails with ENOENT, return 404. Set Content-Length to the file size from fstat().

Stuck? → Hints H3

R4 — MIME Type Mapping

Set Content-Type based on the file extension. Required mappings:

ExtensionContent-Type
.htmltext/html
.csstext/css
.jsapplication/javascript
.txttext/plain
.jpgimage/jpeg
.pngimage/png
(other/none)application/octet-stream
Stuck? → Hints H4

R5 — Subdirectory Serving

Paths like /subdir/page.html must resolve correctly to <docroot>/subdir/page.html. No special handling required beyond correct path concatenation — but verify it works.

R6 — Error Responses

All error responses (400, 403, 404, 405) must still include valid HTTP headers and a short plain-text body describing the error. Carry forward TRD-01 behavior for 400 (malformed) and 405 (wrong method).

Post-Completion Comprehension

  1. Why is checking for .. in the raw URI insufficient to prevent directory traversal?
  2. What does realpath() return if the target file does not exist?
  3. Why use fstat() on an open fd rather than stat() on the path? (Think about race conditions.)
  4. What happens if read() returns fewer bytes than st_size? Is the file corrupt, or is something else going on?
  5. Why does .jpg map to image/jpeg and not image/jpg?
