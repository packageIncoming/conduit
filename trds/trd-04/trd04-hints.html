TRD-04 Hints

Progressive Guidance — Conduit HTTP Server Project
Level 1 = nudge  |  Level 2 = stronger push  |  15+ minutes before escalating

H1 — Epoll Setup and Non-Blocking Sockets (R1, R2)

Level 1

Three setup steps before the event loop: (1) create the epoll instance, (2) set the listen socket to non-blocking, (3) register the listen socket with epoll for EPOLLIN | EPOLLET. The epoll_event.data.fd should be set to the listen fd so you can identify it in the event loop.

Level 2
void set_nonblocking(int fd) {
    int flags = fcntl(fd, F_GETFL, 0);
    fcntl(fd, F_SETFL, flags | O_NONBLOCK);
}

int epfd = epoll_create1(0);
if (epfd == -1) { perror("epoll_create1"); exit(1); }

set_nonblocking(listen_fd);

struct epoll_event ev;
ev.events = EPOLLIN | EPOLLET;
ev.data.fd = listen_fd;
epoll_ctl(epfd, EPOLL_CTL_ADD, listen_fd, &ev);

Note: for the listen socket, data.fd is fine. For client sockets, you'll switch to data.ptr (see H3).

H2 — Edge-Triggered Accept Loop (R4)

Level 1

In edge-triggered mode, one EPOLLIN on the listen socket may signal multiple pending connections. Call accept() in a while(1) loop. When it returns -1 and errno is EAGAIN, all pending connections have been accepted — break out.

Level 2
void handle_accept(int epfd, int listen_fd) {
    while (1) {
        int client_fd = accept(listen_fd, NULL, NULL);
        if (client_fd == -1) {
            if (errno == EAGAIN || errno == EWOULDBLOCK) return;
            perror("accept");
            return;
        }
        set_nonblocking(client_fd);

        connection_t *conn = calloc(1, sizeof(connection_t));
        conn->fd = client_fd;
        conn->state = CONN_READING;

        struct epoll_event ev;
        ev.events = EPOLLIN | EPOLLET;
        ev.data.ptr = conn;
        epoll_ctl(epfd, EPOLL_CTL_ADD, client_fd, &ev);
    }
}

calloc zeroes the struct, so read_len, write_offset, etc. start at 0.

H3 — Per-Connection State Struct (R5)

Level 1

Each connection needs its own buffers and state because the event loop handles many connections interleaved. You can't use a single global buffer. Store a pointer to the connection_t in epoll_event.data.ptr so when epoll fires an event, you can cast events[i].data.ptr back to connection_t * to access that connection's state.

Level 2
typedef enum { CONN_READING, CONN_WRITING, CONN_DONE } conn_state_t;

typedef struct {
    int fd;
    conn_state_t state;
    char read_buf[8192];
    int read_len;
    char *write_buf;       /* malloc'd serialized response */
    int write_len;
    int write_offset;
} connection_t;

In the event loop, recover the connection:

connection_t *conn = (connection_t *)events[i].data.ptr;

But first check if the event is for the listen socket. A simple way: compare events[i].data.fd to listen_fd — but you set data.ptr for clients, not data.fd. One pattern: check if the event's fd matches the listen fd by keeping data.fd for the listen socket only and data.ptr for clients. Differentiate by checking if events[i].data.fd == listen_fd first (this works because the union's memory overlaps — the fd is the first few bytes of the pointer on most systems). A cleaner pattern: wrap the listen socket in its own connection_t with a special state, or simply store the listen fd in a variable and compare.

H4 — Read Handling with Edge-Triggered Drain (R6)

Level 1

When EPOLLIN fires on a client fd, read in a loop. Each iteration appends to read_buf + read_len. After each successful read, check for \r\n\r\n. When found, process the request. If read() returns EAGAIN, stop — all currently available data has been consumed. If read() returns 0, the client disconnected — clean up.

Level 2
void handle_read(int epfd, connection_t *conn) {
    while (1) {
        int space = sizeof(conn->read_buf) - conn->read_len - 1;
        if (space <= 0) {
            /* Buffer full, no \r\n\r\n found → 400 */
            send_error_and_close(epfd, conn, 400);
            return;
        }
        int n = read(conn->fd, conn->read_buf + conn->read_len, space);
        if (n == -1) {
            if (errno == EAGAIN) return;   /* done for now */
            close_connection(epfd, conn);  /* real error */
            return;
        }
        if (n == 0) {
            close_connection(epfd, conn);  /* client disconnected */
            return;
        }
        conn->read_len += n;
        conn->read_buf[conn->read_len] = '\0';

        if (strstr(conn->read_buf, "\r\n\r\n")) {
            /* Full headers received → parse, build response */
            process_request(epfd, conn);
            return;
        }
    }
}

H5 — Write Handling with Partial Send Tracking (R7)

Level 1

The write buffer holds the full serialized response. write_offset starts at 0. Each write attempt starts from write_buf + write_offset with write_len - write_offset bytes remaining. Advance the offset by however many bytes were actually written. When offset equals total length, the response is complete.

Level 2
void handle_write(int epfd, connection_t *conn) {
    while (conn->write_offset < conn->write_len) {
        int remaining = conn->write_len - conn->write_offset;
        int n = write(conn->fd,
                      conn->write_buf + conn->write_offset,
                      remaining);
        if (n == -1) {
            if (errno == EAGAIN) return;   /* buffer full, wait */
            close_connection(epfd, conn);  /* real error */
            return;
        }
        conn->write_offset += n;
    }
    /* All bytes sent */
    close_connection(epfd, conn);
}

When transitioning from READING to WRITING, modify the epoll registration to watch EPOLLOUT | EPOLLET instead of EPOLLIN:

struct epoll_event ev;
ev.events = EPOLLOUT | EPOLLET;
ev.data.ptr = conn;
epoll_ctl(epfd, EPOLL_CTL_MOD, conn->fd, &ev);
Debugging tip: If the server works for single requests but hangs under concurrency, the most likely cause is a missing ET drain loop — either in accept, read, or write. Run strace -f -e trace=network ./conduit <port> <docroot> and look for accept/read/write calls that happen only once per event instead of looping to EAGAIN.
